📋 Formats de Réponse n8n Acceptés

Voici les différents formats de réponse que Flowise.js peut traiter :

✅ Format 1: Standard (Object avec output)

RECOMMANDÉ
{
  "output": "# Résultat\n\n| Colonne 1 | Colonne 2 |\n|-----------|----------|\n| Valeur A | Valeur B |"
}

Configuration n8n:

// Dans le Code Node
return {
  json: {
    output: "votre markdown avec tables"
  }
};

✅ Format 2: Array avec output

ACCEPTÉ
[
  {
    "output": "# Résultat\n\n| Colonne 1 | Colonne 2 |\n|-----------|----------|\n| Valeur A | Valeur B |"
  }
]

Configuration n8n:

// Dans le Code Node
return [
  {
    json: {
      output: "votre markdown avec tables"
    }
  }
];

✅ Format 3: Programme de travail (data)

ACCEPTÉ
[
  {
    "data": {
      "etape_mission_Synthese": [
        {
          "Table_1": {
            "titre": "Mon titre",
            "description": "Ma description"
          }
        },
        {
          "Table_2": [
            {
              "colonne1": "valeur1",
              "colonne2": "valeur2"
            }
          ]
        }
      ]
    }
  }
]

Note: Ce format est automatiquement converti en markdown

✅ Format 4: Tables array

ACCEPTÉ
{
  "tables": [
    {
      "markdown": "| Col1 | Col2 |\n|------|------|\n| A | B |"
    },
    {
      "markdown": "| Col3 | Col4 |\n|------|------|\n| C | D |"
    }
  ]
}

❌ Format Invalide 1: Réponse vide

ERREUR
(rien - 0 caractères)

Erreur: "Réponse n8n vide - vérifiez votre workflow n8n"

Cause: Pas de node "Respond to Webhook" dans le workflow

Solution: Ajoutez un node "Respond to Webhook" à la fin

❌ Format Invalide 2: Texte brut (pas JSON)

ERREUR
Ceci est du texte brut, pas du JSON

Erreur: "Réponse n8n invalide (pas du JSON)"

Cause: Le node retourne du texte au lieu de JSON

Solution: Configurez le node pour retourner du JSON

❌ Format Invalide 3: JSON sans output/data/tables

ERREUR
{
  "result": "quelque chose",
  "status": "ok"
}

Erreur: "Format de réponse non reconnu"

Cause: Pas de clé "output", "data" ou "tables"

Solution: Utilisez une des structures acceptées ci-dessus

🔧 Workflow n8n Minimal

Voici un workflow minimal qui fonctionne :

Structure du Workflow

1. Webhook (POST /webhook/htlm_processor)
   ↓
2. Code Node
   ↓
3. Respond to Webhook

Code Node

// Récupérer les données
const input = $input.first().json.question;

// Traiter et créer le markdown
const result = `# Résultat du Traitement

| Colonne 1 | Colonne 2 |
|-----------|-----------|
| Valeur A  | Valeur B  |
| Valeur C  | Valeur D  |`;

// Retourner au format attendu
return {
  json: {
    output: result
  }
};

Respond to Webhook

Response Mode: Using 'Respond to Webhook' Node
Response Body: {{ $json }}
